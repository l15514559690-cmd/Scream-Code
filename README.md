## 下载

用 Git 克隆本仓库，或下载源码 ZIP 后解压到本地文件夹。

## 安装

在**项目根目录**执行：

```bash
bash install.sh
```

## 配置

复制环境变量模板并填写 API 等信息：

```bash
cp .env.example .env
```

用文本编辑器打开 `.env`，按说明填入密钥与模型地址。

## 启动

安装脚本执行完后，按终端提示执行 `source ~/.zshrc`（或脚本给出的配置文件路径）。然后在终端输入：

```bash
scream
```

## ⚡ 强大的斜杠指令库

Scream Code 内置了极其丰富的系统指令，在输入框输入 `/` 即可触发（Rich 行内 REPL 与全屏 TUI 均支持透传；**完整列表以终端内 `/help` 为准**）。配置模型也可用 **`scream-config`**。

#### 🐺 核心干活指令

* `/team` —— **【群狼模式开关】**。开启后，任务将由 Planner（规划师）→ Coder（程序员）→ Reviewer（审查员）协同完成，适合处理复杂大需求。另支持 **`$team <提示>`**：仅单条消息走团队编排，不改变开关。
* `/stop` —— **【紧急刹车】**。AI 输出过长或工具链卡住时，立即请求中断当前轮的生成与后续 tool 调用（长 bash 等会尽快结束）。
* `/diff` —— **【一键看改动】**。展示当前工作区的 **`git status --short`** 与 **`git diff --stat`** 摘要，快速把握已修改文件与变更规模。

#### 🧠 记忆与上下文管理

* `/memo [内容]` —— **【长效记忆】**。输入 `/memo <内容>` 直接写入项目记忆（SCREAM.md / CLAUDE.md）；若**不带参数**输入 `/memo`，会由模型根据当前会话自动提炼要点并写入（需已启用 LLM 的 REPL）。
* `/summary` —— **【生成总结】**。输出当前工作区与会话的 Markdown 摘要，并可选择是否追加到永久记忆。
* `/new` —— **【彻底重置】**。硬重置：新 `session_id`、清空对话与计数、落盘新会话；比普通清屏更彻底（**不删除**已写入的长效记忆文件）。
* `/flush` —— **【本轮清空】**。清空本轮对话、重置 token 累计并落盘新会话，比重置更轻量。
* `/sessions` —— **【历史记录】**。扫描 `.port_sessions`（及工作区索引）列出历史聊天会话。
* `/load <会话ID>` —— **【恢复聊天】**。配合 `/sessions` 使用，加载指定 id 的会话上下文继续对话。

#### 🩺 系统与账单监控

* `/cost` —— **【查看账单】**。查看当前会话累计的 Input/Output Token 与粗略费用示意（以厂商账单为准）。
* `/doctor` —— **【系统体检】**。检查 Python 依赖、路径、权限与关键配置是否就绪。
* `/status` —— **【运行状态】**。查看当前模型、沙箱/全局权限、工具加载、`.claw.json`、项目记忆路径等。
* `/help`（或 `/?`）—— **【百宝箱】**。在终端内呼出分组帮助（与下述开发者命令说明一致）。

> **全屏 TUI** 下还可使用 **`/exit`** 或 **`/quit`** 正常退出界面（由 Rust 前端处理）。

（注：还有 **`/audit`**、**`/report`**、**`/subsystems`**、**`/graph`** 等面向开发者的硬核调试命令，可通过 **`/help`** 在终端内探索。）
